1. Product Overview & Philosophy

SchoolConnect is a WhatsApp-first school communication and management platform built for Indian schools. The core insight: every teacher and parent in India already uses WhatsApp. SchoolConnect meets them where they are — no app to download, no login to remember, no training required.

1.1 The Core Problem

Indian schools (especially Tier 2/3 cities) struggle with:

  • Teachers spending 30–45 minutes/day on manual parent communication (calling, WhatsApp groups)
  • Parents missing important updates buried in noisy class WhatsApp groups
  • No reliable attendance tracking or parent notification system
  • School management software that requires training and is rarely used
  • Fee reminders sent manually, leading to delayed collections
  • No audit trail — disputes about whether a message was sent

1.2 The SchoolConnect Solution

A teacher types one natural language message on WhatsApp. SchoolConnect's AI understands it, formats it professionally, and delivers it to the right parents instantly. No forms, no portals, no training.

📱 For Teachers
  • Send any message in plain language
  • AI understands and routes automatically
  • No app, no login — just WhatsApp
  • Saves 30–45 min/day per teacher
  • Instant delivery confirmation
👪 For Parents
  • Personalized, professional updates
  • Reply to attendance alerts (SICK/KNOWN)
  • Ask questions in natural language
  • No app to download
  • Works on any phone with WhatsApp
🏫 For Principals
  • Real-time dashboard of all communications
  • Attendance reports by class/student
  • Fee collection tracking
  • Delivery analytics (sent/failed/read)
  • School-wide broadcast capability
🏢 For School Owners
  • Professional image for the school
  • Reduced admin staff workload
  • Better parent satisfaction & retention
  • Data-driven decisions
  • Full audit trail & compliance

The teacher interface is the heart of SchoolConnect. Teachers interact entirely through WhatsApp — the same app they already use. No training, no new software, no login.

2.1 Natural Language Message Input ✓ Live

Teachers type messages exactly as they would to a colleague. The AI understands intent, extracts data, and takes action. Messages can be in English or Hinglish.

2.4 Smart Student Matching ✓ Live

Teachers don't need to type exact names. 3-tier fuzzy matching:

  1. Exact match — "Diksha" matches "Diksha Sharma"
  2. Partial/first name — "Dik" matches "Diksha Sharma"
  3. Parent name — "Meena" matches student whose parent is "Meena Sharma"

If no match: teacher receives a helpful list of all students in the class.

2.5 Multi-Class Teacher Support ✓ Live

A teacher can be assigned to multiple classes. If no class is specified, the system uses the teacher's primary class. Teachers can always override by specifying the class explicitly.

2.6 Delivery Confirmation ✓ Live

After every action, the teacher receives an instant WhatsApp confirmation: student/class name, number of parents notified, failures (if any), and timestamp.

2.7 Scheduled Messages ▶ MVP

The scheduler runs every 5 minutes and processes notifications with a future scheduledAt time. UI for scheduling is planned (e.g., "Send fee reminder tomorrow 8am").

2.8 Hinglish / Regional Language Input ▶ Partial

The AI understands Hinglish messages like "Rahul nahi aaya class 8B" or "kal school band hai". Language preference stored per parent (EN/HI/PA). Full regional language output to parents is planned.

2.9 Unregistered Teacher Handling ✓ Live

If a message comes from an unregistered number, the system responds: "Your number is not registered in SchoolConnect. Please contact your school admin." No action is taken.

3. AI Message Engine

SchoolConnect uses Claude (Anthropic) as its AI backbone. The AI does two jobs: (1) understand what the teacher means, and (2) write a professional, warm message for parents.

3.1 Intent Classification ✓ Live

Every teacher message is classified into one of 7 intents. The AI returns a confidence level (HIGH/MEDIUM/LOW). Low confidence triggers a clarification prompt to the teacher rather than sending a wrong message.

3.2 Structured Data Extraction ✓ Live

From a single natural language message, the AI extracts: intent, student name, class identifier (e.g. "8B"), subject, date (ISO format from relative dates like "Friday", "tomorrow", "25 April"), and confidence level.

3.3 Automatic Parent Message Formatting ✓ Live

The AI writes the parent-facing message automatically: professional and warm tone, under 150 words, starts with a relevant emoji, includes all relevant details, contextually appropriate for the message type.

Teacher types: "Math test Friday chapters 4-5 grade 8B" → Parent receives:
📚 Test Reminder — Class 8B

Dear Parent,

This is a reminder that your child has a Mathematics test on Friday. The test will cover Chapters 4 and 5.

Please ensure your child revises the topics and is well-prepared. Wishing them all the best! 🌟

— SchoolConnect

3.4 Clarification Handling ✓ Live

When the AI is unsure (LOW confidence or UNKNOWN intent), it asks the teacher to clarify with specific options. This prevents wrong messages from being sent to parents.

3.5 AI Model Flexibility ✓ Live

Currently uses Claude Sonnet 4.5. The system is model-agnostic — can switch to GPT-4o-mini or Gemini Flash for cost optimization without changing application logic.

3.6 Parent AI Assistant ◆ Planned

Parents will ask questions in natural language via WhatsApp: "How many days was Rahul absent this month?", "What is the fee due?", "When is the next PTM?", "What homework is due tomorrow?". The AI queries the database and responds in the parent's preferred language. Cost: ~₹0.017/query.

3.7 AI-Powered Report Summaries ◆ Planned

Monthly AI-generated summaries for principals: attendance trends, top absent students, fee collection status, communication effectiveness metrics.

3.8 Smart Duplicate Detection ◆ Planned

Detect if a teacher accidentally sends the same message twice within 5 minutes and ask for confirmation before sending again. Prevents duplicate notifications to parents.

4. Attendance Management

Attendance is SchoolConnect's flagship feature. A teacher marks a student absent in 5 seconds via WhatsApp. The parent is notified instantly. The parent can reply to explain the absence.

4.1 WhatsApp Attendance Marking ✓ Live

Teacher sends "Diksha absent class 8B". System: identifies teacher → finds student (fuzzy match) → finds parent's WhatsApp → sends AI-formatted alert → logs record → confirms to teacher. All in under 3 seconds.

4.2 Parent Absence Reply System ✓ Live

4.3 Attendance Log & History ▶ MVP

Every absence stored with: student, date, teacher, parent notification timestamp, parent reply. Admin API exposes last 50 logs. Full dashboard with filtering/search is planned.

4.4 Bulk Attendance Marking ◆ Planned

Teacher sends: "Absent today 8B: Rahul, Priya, Amit". System processes all students in one message, sends individual alerts to each parent, confirms with a summary to the teacher.

4.5 Daily Attendance Summary ◆ Planned

Automated daily report to principal at end of school day: total present/absent per class, names of absent students, parent reply status (SICK/KNOWN/no reply/disputed).

4.6 Monthly Attendance Report ◆ Planned

Monthly PDF per class: attendance % per student, trend charts, students below 75% threshold flagged for action. Sent to principal via WhatsApp and email.

4.7 Attendance Dispute Resolution ◆ Planned

When a parent replies PRESENT (disputes absence), the system flags it for the teacher/principal to review. Teacher can correct the record. Parent is notified of the correction.

4.8 Late Arrival Tracking ◆ Planned

Teacher sends "Rahul late 8B". System logs late arrival, notifies parent: "Your child Rahul arrived late today at [time]. Please ensure timely arrival."

4.9 Leave Application via WhatsApp ◆ Planned

Parent sends a leave request via WhatsApp: "Priya will be absent tomorrow for a family function". System logs the pre-approved leave, notifies the teacher, and marks it as KNOWN absence automatically.

4.10 Attendance Analytics ◆ Planned

Dashboard showing: class-wise attendance heatmap, students with chronic absenteeism, day-of-week patterns (which day has most absences), month-over-month trends.

5. Notification & Broadcast System

SchoolConnect's broadcast engine delivers personalized WhatsApp messages to hundreds of parents in seconds. Every message is tracked per-parent for delivery status.

5.1 Class-Level Broadcast ✓ Live

Send a notification to all opted-in parents in a specific class. Teacher specifies class (e.g., "8B") and the message goes to all parents of students in that class only.

5.2 School-Wide Broadcast ✓ Live

Send a notification to all opted-in parents across all classes in the school. Used for emergencies, school events, and general announcements. Teacher says "all classes" or sends an EMERGENCY type message.

5.3 Per-Parent Delivery Tracking ▶ MVP

Every message sent to every parent is logged in MessageLog with: Twilio message SID, delivery status (QUEUED/SENT/DELIVERED/FAILED), sent timestamp. Teacher sees summary (X sent, Y failed). Full per-parent status in dashboard is planned.

5.4 Opt-In / Opt-Out ▶ MVP

Parents can opt out of notifications. The optedIn flag is stored per parent. Only opted-in parents receive broadcasts. Opt-out via WhatsApp reply ("STOP") is planned.

5.5 Notification Types ✓ Live

TypeUse CaseTarget

TEST_REMINDERUpcoming test/examClass parents

HOMEWORKHomework assignmentsClass parents

5.6 Scheduled Notifications ▶ MVP

Notifications can be scheduled for future delivery. The scheduler checks every 5 minutes for pending notifications whose scheduled time has passed and sends them. Useful for: morning reminders, fee due date alerts, exam day reminders.

5.7 Notification History & Audit Trail ▶ MVP

Every notification stored permanently with: type, raw teacher message, formatted parent message, target class, school, teacher, scheduled time, sent time, status, recipient count. Full audit trail for compliance.

5.8 Failed Message Retry ◆ Planned

When a WhatsApp message fails (invalid number, WhatsApp not installed), the system logs the failure and retries automatically after 30 minutes. Admin can see all failed deliveries and manually retry.

5.9 Message Templates ◆ Planned

Pre-approved WhatsApp message templates for common scenarios: monthly fee reminder, exam schedule, holiday notice, PTM invitation. Teachers select a template and fill in variables.

6. Parent WhatsApp Experience

Parents receive personalized, professional notifications directly on WhatsApp. No app to download, no account to create. Designed to be simple enough for any parent regardless of tech literacy.

6.1 Personalized Notifications ✓ Live

Every message sent to a parent is personalized with their child's name. Parents never receive generic group messages — every notification feels like it was written specifically for them.

6.2 Absence Alert with Reply Options ✓ Live

When a parent receives an absence alert, they reply with SICK, KNOWN, or PRESENT. The system responds appropriately and logs the reply. No forms, no portals — just a one-word reply.

6.3 Language Preference ▶ Partial

Language preference stored per parent: English (EN), Hindi (HI), Punjabi (PA). Currently all messages sent in English. Hindi and Punjabi message generation planned using the same AI pipeline.

6.4 Opt-Out / Opt-In Control ▶ Partial

Parents can opt out of notifications. Opt-out flag stored in database. WhatsApp STOP/START command handling is planned. Parents who opt out will not receive broadcasts but will still receive individual attendance alerts.

6.5 Parent Self-Service Commands ◆ Planned

Parents send commands to get information without calling the school:

  • ATTENDANCE — This month's attendance summary for their child
  • FEES — Current fee balance and due date
  • TIMETABLE — Today's class schedule
  • HOMEWORK — Pending homework for their child's class
  • RESULTS — Latest exam results
  • CONTACT — School contact details

6.6 Fee Payment via WhatsApp ◆ Planned

Parents receive a fee reminder with a payment link embedded in the WhatsApp message. They pay directly via UPI/Razorpay without leaving WhatsApp. Payment confirmation sent automatically.

6.7 Parent Feedback ◆ Planned

After PTM or school events, parents receive a 1-question feedback request via WhatsApp: "How would you rate today's PTM? Reply 1-5". Responses aggregated for principal dashboard.

Automated WhatsApp reminders sent to parents with unpaid fees: 7 days before due date, on due date, 3 days after due date. Message includes amount due, due date, and payment link. Stops automatically when fee is paid.

15. Platform & Technical Features

15.1 WhatsApp Integration ✓ Live

Currently via Twilio WhatsApp API (MVP). Planned migration to Interakt (Meta-approved BSP) for production. Interakt provides lower per-message costs, better deliverability, and a shared inbox dashboard.

15.2 Multi-Tenant Architecture ▶ MVP

Single deployment serves multiple schools. Each school's data is isolated by schoolId. One Railway deployment, one database, multiple schools. Cost scales efficiently.

15.3 Notification Scheduler ▶ MVP

Cron-based scheduler runs every 5 minutes. Processes pending notifications with past scheduledAt time. Handles failures gracefully with status updates. Designed to be idempotent.

15.4 Error Handling & Monitoring ▶ MVP

All errors logged with context. Failed WhatsApp sends logged in MessageLog. Notification status updated to FAILED on error. Sentry integration planned for production error tracking.

15.5 Security ▶ MVP

Teacher authentication by phone number (only registered numbers can send commands). Webhook signature verification (Twilio). Environment variables for all secrets. HTTPS enforced on Railway and Vercel.

15.6 Scalability ▶ MVP

Broadcasts use Promise.allSettled for parallel sending — 500 parents notified in ~2-3 seconds. Railway auto-scales on demand. Database connection pooling via Prisma.

15.7 File Storage ◆ Planned

Cloudflare R2 for storing: student profile photos, uploaded circulars/PDFs, exported reports, fee receipts. Zero egress fees. Free tier covers first 10GB (sufficient for 100+ schools).

15.8 Email Notifications ◆ Planned

Resend for transactional emails: OTP verification, monthly reports to principals, fee invoices, welcome emails for new schools. Free tier (3,000 emails/mo) covers initial stage.

15.9 Redis Cache ◆ Planned

Upstash Redis for: rate limiting AI queries, session caching, message queue for bulk broadcasts. Free tier (10,000 commands/day) covers initial stage.
